# 亚马逊供应链智能决策系统 · 使用说明

基于第5周需求预测与第6周库存优化的分析成果，使用 Streamlit 构建的交互式决策仪表盘。

## 一、环境要求

pip install streamlit pandas numpy scipy pulp

## 二、启动方式

在项目根目录执行：

    streamlit run dashboard/app.py

启动后浏览器自动打开（默认 http://localhost:8501）。停止运行按 Control + C。

## 三、目录结构

    dashboard/
    ├── app.py                  主入口（系统介绍 + 导航）
    ├── pages/
    │   ├── 1_销售概览.py        销售趋势、品类占比、节假日对比
    │   ├── 2_需求预测.py        历史+预测趋势、各品类预测精度
    │   ├── 3_库存优化.py        ABC分类、安全库存、EOQ（实时计算）
    │   └── 4_补货计划.py        12周补货计划（PuLP实时求解）
    ├── utils/
    │   └── data_loader.py      统一数据加载（带缓存）
    └── README.md               本文档

## 四、四个页面功能

### 页面1 · 销售概览
解决问题：现在卖得怎么样？
控件：品类多选、日期范围。展示：销售额/销量/SKU数/日均销量、日销量趋势、品类占比、节假日对比。

### 页面2 · 需求预测
解决问题：未来会卖多少、预测有多准？
控件：品类选择、历史周数、预测周数。展示：预测精度(WMAPE)、历史+预测合并趋势图、各品类精度对比。

### 页面3 · 库存优化（核心）
解决问题：该备多少货？服务水平与成本如何权衡？
控件：品类、服务水平、持库率、提前期、订货成本。
特性：拖动滑块，安全库存、缺货成本、EOQ 实时重算。

### 页面4 · 补货计划（核心）
解决问题：什么时候订、订多少？
控件：品类、服务水平、订货成本、持库率、期初库存。
特性：输入实际期初库存，PuLP 实时重新求解最优补货计划，实现滚动优化。

## 五、数据来源

仪表盘读取 data/processed/ 下的 CSV，全部来自第5、6周分析：
- amazon_daily_sales_train.csv（销售明细，页面1）
- forecast_results.csv / forecast_accuracy.csv（需求预测，页面2）
- abc_classification.csv（ABC分类，页面3）
- replenishment_plan_3months.csv（补货计划基准，页面4）

## 六、典型使用场景

每周补货决策：
1. 进入「补货计划」页面，选择品类
2. 在「期初库存」输入框填入仓库当前实际库存
3. 系统实时给出未来12周的最优订货计划
4. 查看「首次订货」指标，判断本周是否需要下单

## 七、注意事项

- 页面4每次调参会调用 PuLP 求解，需等待 1~2 秒
- 首次加载数据后会缓存，后续切换页面更快
- 修改参数后图表自动刷新，无需手动操作
